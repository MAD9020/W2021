# Final project: Grocery store self-checkout

# Details

Your final project involves the design of a new touch screen interface for grocery store self-checkout.

Your deliverables are a prototype (to be submitted) AND an in-class presentation.

# The context:

Many stores now provide self-checkout lanes, where customers can complete their purchases using a touch-screen interface, a scanner, and weight-sensitive packing surfaces. While the basic interaction steps of beginning the checkout, scanning individual items, and making a payment are supported, many customers experience confusion, inefficiency, and hurdles when completing their transaction. There is much opportunity for an improved checkout experience for guests, using some fundamental principles of good UI and UX design.

Field Study: You will observe customers checking out the existing systems, and find one major difficulty that is being experienced by users. This difficulty will map to a Usability Heuristic. You will improve the checkout process by providing a solution to this difficulty.

REFERENCE Material: Due to COVID-19 pandemic, please use these resources instead:

  • field study photos
  • explaining video

# For the purpose of this project, we're making the following assumptions:

  • You have been hired by a large grocery chain to design an app
  • The facility is planning to install tablets in the place of existing screens. They will be attached, fixed permanently, in landscape orientation, and configured to run this app only. (Customers can't exit the app to do anything else with the tablet.)
  • Existing scanners, voice prompts and weight-sensitive packing surfaces will continue to be used.

# Requirements

Your self-checkout app must include the following functionality:

  • Initial screens, which support the following functionality:
    • Welcome/start screen, which customers will see when they approach the checkout.
    • Add your own grocery bags
    • Change language (French/English)
    • Any alert messages required by store
    • Start checking out items
  • Scanning/entering items screens, which support the following functionality:
    • Scan bar codes
    • Enter products without barcode (produce/bakery items) using code or search/browse
    • See list of products added
    • Remove item
    • Call attendant
    • Handling of errors/alerts/messages (ex. put item in bag, invalid code, etc.)
  • Complete transaction screens, which support the following functionality:
    • Indicate number of bags used
    • Scan points card
    • Select payment method
    • Error handling
    • Confirmation
  • The screen(s) supporting the specific problem you're trying to improve
    • Show where in the checkout the problem occurs
    • Show the screen(s)/elements on screen/voice instructions/etc that will improve the problem you observed

Things to consider

  • What's the best way to present the initial screens? Is it one screen? A wizard? Is there a 'quick start' shortcut? Something else?
  • Once guests are checking out items, do they have the necessary guidance and feedback systems? What are their choices? What must be on the screen?
  • When adding an item without a barcode, does the system help users know what to do? Do you use appropriate controls/prompts?
  • For optional paths (like, adding bags, or entering a points card, etc.), how do you smoothly support users who DO use them, without annoying those who DON'T?
  • When ready to pay, what options must be available on-screen?
  • Which screens require a 'back' or 'exit' functionality? Which ones don't?
